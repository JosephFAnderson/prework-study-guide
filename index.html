HTML

  • !DOCTYPE declartion tells the browser what type of document it should expect
  • The head element is a container for metadata
  • Meta elements contain information about the page that is used by the browser
  • The user does not see information in the head or meta elements
  • Types of information that can be stored in the meta tag
  • The body element represents the visible content shown to the user
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage
  • When possible, it is important to use semantic HTML elements to provide easier reading for other devolopers and the browser
  • Semantic HTML elements also make your webpage accesible to the widest variety of users, including those who use assistive technologies such as screen readers
  • Elements nested inside other elements are called child elements

CSS

  • Cascading Style Sheets (CSS) is a programming language used by devolopers to define how webpager content is presented to users
  • CSS defines the colors, aesthetics, visual layout, and organization
  • There are 3 ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • Inline CSS is where you style an element directly in the HTML File
  • Inline CSS applies to a particular page element. Thus they are quick but less flexible
  • Internal style sheet is where you embed an entire stylesheet directly in the HTML file using a style element in the head element
  • The internal style sheet contains CSS rules for the webpage that apply to the page. Thus allow you to style multiple elements at once
  • External style sheet are a seperate file that are referenced with a link element palced in the HTML's head element
  • External style sheets are the most common method as it allows devolopers to keep all their CSS rules in a seperate file (Seperation of concerns)
  • Seperation of concerns refers to separating code into different files based on use. Wiki article
